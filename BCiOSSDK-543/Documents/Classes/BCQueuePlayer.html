Overview

BCQueuePlayer manages a queue of videos to play sequentially. It can mostly be used as a drop-in replacement for an AVQueuePlayer but it will actually attempt to preload videos in the queue and won’t collapse if you attempt to play videos of different formats with it. It also has convenience methods for playing BCVideos and BCPlaylists.

playbackEmitter

The primary BCEventEmitter used by the BCQueuePlayer.

@property (nonatomic, readonly) id<BCEventEmitterProtocol> playbackEmitter

Discussion

The event emitter communicates all the time-dependent state of the player. If you would like to perform custom logic when an error occurs during playback, for instance, you could add a listener to the emitter for BCEventErrors. All events within the player that are allowed to propagate can be captured and reacted to through this emitter. For a full listing of event types and their structure, refer to the documentation for the player components.

view

@property (nonatomic, readonly) UIView *view

Discussion

A player is configured by default to resize its subviews when it is resized. You can resize the player view as you would any other UIView:

player.view.frame = CGMakeRect(0, 0, 640, 480);

For more information on modifying the presentation of a player, see the View Programming Guide for iOS
